## Компоненты

| Файл                | Роль                                                        |
| ------------------- | ----------------------------------------------------------- |
| `client/index.html` | UI + вся логика инициализации, подписки, формы, отображения |
| `client/sw.js`      | Service Worker: приём push, показ уведомления, postMessage  |

Взаимодействие между компонентами — только через `postMessage` / `navigator.serviceWorker`.
Никакого общего состояния, никаких глобальных переменных между файлами.

## Поток данных

```
Пользователь
     │
     ▼
index.html ── GET /api/key ──▶ Сервер ──▶ VAPID public key ──▶ index.html
     │
     │  оформить push-подписку
     │
     ▼
index.html ── POST /api/notifications ──▶ Сервер
                                           │ (по расписанию)
                                           ▼
                                      Push Service
                                           │ зашифрованный push
                                           ▼
                                         sw.js
                                           │ показать уведомление
                                           │ сообщить странице
                                           ▼
                                       index.html
                                           │
                                           ▼
                                    Список "Получено"
```

## Проверяемые API

| API               | Назначение                                                  |
| ----------------- | ----------------------------------------------------------- |
| Service Worker    | фоновая обработка push-событий                              |
| Push API          | оформление push-подписки                                    |
| Notifications API | показ системных уведомлений                                 |
| iOS версия        | разграничение: предложить добавить на экран или обновить ОС |
| PWA-режим         | на Apple — определяет доступность Push API                  |

## Ключевые решения

**JS inline в `index.html`, без сборки**
Клиент — инструмент ручного тестирования, не продакшн-приложение. Сборка и зависимости создают overhead без пользы.

**SW — единственный получатель push**
Push-событие доступно только в Service Worker. Страница получает данные через postMessage — это стандартный и единственный путь.

**Состояние в памяти, не в localStorage**
Список "Получено" не нужно персистировать между сессиями — это тестовый инструмент. In-memory проще и не создаёт побочных эффектов.

**URL API из `window.location.origin`**
Клиент и сервер всегда на одном хосте, хардкодить порт не нужно. Работает на любом окружении без конфигурации.

**Совместимость Apple — через версию iOS, не через фича-детект**
`PushManager` в iOS Safari недоступен вне standalone-режима даже на iOS ≥ 16.4. Поэтому нельзя отличить "нет поддержки" от "не в PWA" через один API — нужна комбинация: Apple + версия + standalone.
