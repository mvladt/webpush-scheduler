## Цель

Создать минимальный браузерный клиент для ручного тестирования и демонстрации
полного цикла Web Push уведомлений: подписаться → запланировать → получить.

---

## Сценарии поведения

**Сценарий 1a: Инициализация — всё поддерживается**
- Given: пользователь открывает страницу
- When: Service Worker + Push API + Notifications API — все доступны
- Then: страница инициализируется и отображает статус каждого шага

**Сценарий 1b: Apple устройство без PWA-режима (iOS ≥ 16.4 / macOS Ventura+)**
- Given: пользователь на iOS ≥ 16.4 / iPadOS ≥ 16.4 / macOS Safari с поддержкой PWA push
- When: сайт открыт в браузере, не добавлен на главный экран / в Dock
- Then: Push API недоступен
- Then: страница показывает инструкцию:
  - iOS/iPadOS: «Нажмите "Поделиться" → "На экран Домой"»
  - macOS Safari: «Нажмите "Файл" → "Добавить в Dock"»
- Then: дальнейшая работа невозможна до выполнения инструкции

**Сценарий 1c: Несовместимое устройство / браузер**
- Given: пользователь открывает страницу
- When: один или несколько API недоступны И не применим сценарий 1b
- Then: страница показывает список конкретно недоступных API
- Then: показывается подсказка:
  - Apple устройство: «Обновите iOS / macOS до последней версии»
  - Иначе: «Обновите браузер»
- Then: дальнейшая работа невозможна

**Сценарий 2: Запрос разрешения**
- Given: разрешение на уведомления не выдано
- When: пользователь нажимает кнопку "Разрешить уведомления"
- Then: браузер показывает системный диалог запроса разрешения
- When: пользователь отклоняет
- Then: страница показывает инструкцию как разрешить в настройках браузера

**Сценарий 3: Планирование уведомления**
- Given: подписка активна, разрешение выдано
- When: пользователь вводит текст и дату/время, нажимает "Запланировать"
- Then: уведомление отправляется на сервер с текущей push-подпиской
- Then: страница подтверждает успех или показывает ошибку
- When: поля не заполнены
- Then: форма не отправляется

**Сценарий 4: Получение уведомления (вкладка открыта)**
- Given: уведомление запланировано, вкладка открыта
- When: сервер отправляет push в указанное время
- Then: браузер показывает системное уведомление
- Then: на странице в секции "Получено" появляется запись с текстом и временем получения

**Сценарий 5: Получение уведомления (вкладка закрыта)**
- Given: уведомление запланировано, вкладка закрыта (SW активен в фоне)
- When: сервер отправляет push
- Then: браузер показывает системное уведомление
- Then: при следующем открытии вкладки список "Получено" пуст (данные in-memory, не персистентны)

---

## Граничные случаи

- Сервер недоступен при открытии → показать ошибку получения VAPID-ключа
- Сервер вернул не 2xx при планировании → показать ошибку с HTTP-кодом
- Подписка устарела (410 от push-сервиса) → ошибка логируется, не блокирует UI

---

## Ограничения

- Без фреймворков и систем сборки — только статические файлы
- Работает на `localhost` (HTTPS не требуется для Service Worker)
- Не требует отдельного запуска — файлы отдаются тем же сервером
- URL API не захардкожен — берётся из `window.location.origin`
- Стилизация минимальная: браузерные дефолты + самый необходимый минимум
- На Apple устройствах клиент работает только в PWA-режиме и требует iOS ≥ 16.4 / macOS Ventura+
