# Автоматическая настройка окружения при старте

## Проблема

Приложение требует ручной настройки перед первым запуском: нужно выполнить `npm run setup`, проверить `.env`, указать `VAPID_SUBJECT`. Если пользователь просто делает `npm start` без `.env` — Node.js падает из-за флага `--env-file .env`, не дойдя до кода приложения.

При этом e2e тесты проходят без `.env`, потому что `createTestApp()` генерирует VAPID-ключи самостоятельно. Это создаёт ложное ощущение работоспособности: тесты зелёные, а `npm start` на чистом клоне — не работает.

## Желаемое поведение

`npm ci && npm start` — и всё работает. Без ручных шагов, без предварительной генерации ключей, без `.env.example`.

## Что должно происходить при старте

1. Попытаться загрузить `.env` через `process.loadEnvFile()`
2. Если файл не найден — сгенерировать VAPID-ключи через `web-push`, записать `.env` с дефолтными значениями, загрузить его
3. Если файл найден, но в нём нет VAPID-ключей — аналогично: сгенерировать недостающие, дописать в файл
4. Продолжить обычный запуск

## Риски

- **Потеря подписок при пересоздании ключей.** Если `.env` удалят — приложение сгенерирует новые ключи, и все существующие push-подписки станут невалидными. При генерации новых ключей нужно выводить предупреждение в лог.
