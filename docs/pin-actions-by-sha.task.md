# Задача: закрепить GitHub Actions по SHA вместо тега

## Проблема

Все `uses:` в `.github/workflows/*.yml` (`ci.yml`, `deploy.yml`) указаны по мутабельному тегу
(`@v4`), а не по полному SHA коммита. Тег может быть переиспользован владельцем экшена (умышленно
или после компрометации аккаунта/репозитория) — supply-chain риск: то, что запускается в CI,
не обязательно совпадает с тем, что было при последнем ревью.

## Желаемое поведение

- Все `uses: actions/...@v4` заменены на `uses: actions/...@<полный SHA>  # vX.Y.Z` (комментарий с
  версией — для читаемости).
- Dependabot (`.github/dependabot.yml`, экосистема `github-actions`) сам открывает PR при выходе
  новой версии — SHA обновляется через обычный review/merge PR, а не руками.

## Риски

- Без Dependabot ручной pin-by-SHA — трудноподдерживаемый ад (никто не будет руками отслеживать
  новые версии трёх экшенов). Именно поэтому задача была отложена до появления Dependabot
  (см. `docs/cicd.plan.md`, этап 4.5) — теперь он есть, можно делать.
- Минимальная практическая польза для соло-проекта с низким риском атаки на конкретно эти
  публичные экшены (`actions/checkout`, `actions/setup-node`, `actions/upload-artifact` — от
  самого GitHub, не сторонние). Приоритет ниже, чем у остального в проекте.
